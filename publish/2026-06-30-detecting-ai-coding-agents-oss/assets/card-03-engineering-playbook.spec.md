# Card 03 — Engineering Playbook

## Purpose
Create a practical takeaway visual card for the section `给我们的启示`.

## Card title
让 AI Coding 进入供应链，但必须可见

## Core message
The governance goal is not to block AI coding agents; it is to make agent participation attributable, reviewable, and measurable.

## Required content
Show five checklist items:
1. `Multi-signal measurement` — PR + commit + config + CI + review
2. `AI attribution metadata` — agent, model, human owner, change source
3. `Greenfield / Legacy split` — scaffold vs migration playbook
4. `Quality metrics` — review effort, revert, defects, tests, incidents
5. `Evidence chain` — done criteria, test proof, rollback path

## Suggested layout
A vertical engineering checklist beside a simplified pipeline:
`Agent Change → Human Review → CI Evidence → Release → Observability Feedback`
