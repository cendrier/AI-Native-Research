# Visual Card Spec: Agent Control Rollout Checklist

## Card id

card-04-rollout-checklist

## Insert location

After paragraph containing: `短期落地可以从四件事开始`

## Purpose

Give readers a practical checklist they can apply before expanding AI coding agents into more repositories and higher-risk workflows.

## Core message

Before scaling coding agents, make sure the organization can identify, limit, observe, intercept, recover, and improve agent behavior.

## Layout

Use a 16:9 checklist card with six grouped boxes.

Title: `Before Scaling Coding Agents`

Box 1: `Identity` — separate agent actor; attribution; revocation

Box 2: `Permission` — least privilege; risk-based access; secrets boundary

Box 3: `Runtime` — sandbox; separated scaffold/commands; network policy

Box 4: `Telemetry` — task goal; tool calls; file writes; PR diff; test logs

Box 5: `Control` — monitor; block; escalate; rollback; shutdown

Box 6: `Metrics` — coverage; recall; time-to-response; incident rate

Bottom note: `Do not scale autonomy faster than control.`

## Visual style

Follow `docs/AI_NATIVE_RESEARCH_VISUAL_STYLE_V1.md`. Make it a crisp operating checklist, not a poster. Use subtle hierarchy and clean cards.
