# Visual Card Spec: Detection and Response Ladders

## Card id

card-02-detection-response-ladders

## Insert location

After heading: `核心洞察`

## Purpose

Summarize the two capability ladders from the GDM AI Control Roadmap: Detection D1-D4 and Prevention/Response R1-R3. This is the main framework card for the article.

## Core message

As agents become harder to oversee and more capable of harm, defenses must escalate from transcript monitoring to behavior monitoring, real-time access control, strict supply-chain security, and shutdown capability.

## Layout

Use a clean 16:9 card with two side-by-side ladders.

Left ladder title: `Detection: D1 → D4`.

D1: `Monitorable CoT` → `CoT monitoring + alignment tests`.

D2: `Oversight aware` → `robust monitors + realistic tests`.

D3: `Opaque reasoning` → `behavior / PR / anomaly monitoring`.

D4: `Monitor collusion risk` → `internals-based monitoring`.

Right ladder title: `Response: R1 → R3`.

R1: `Low-severity reversible harm` → `async alerts + investigation`.

R2: `Can achieve TRAIT&R objectives` → `real-time block + isolation + fine-grained access`.

R3: `Cyber advantage over trusted defenders` → `supply-chain hardening + shutdown capability`.

Bottom band: `Operating metrics: coverage · recall · time-to-response · response coverage`.

## Visual style

Follow `docs/AI_NATIVE_RESEARCH_VISUAL_STYLE_V1.md`. Use structured research-card design, not a decorative infographic. The ladders should feel like an operating model rather than a risk poster.

## Text constraints

Keep each row concise. Prefer aligned labels and simple arrows. Ensure the card is readable on Feishu mobile.
