# Visual Card 02 — 一个成熟 SKILL.md 应该包含什么

## Purpose

Turn the paper's six SKILL.md content themes into an internal checklist that teams can use when authoring or reviewing agent skills.

## Layout

Create a central `SKILL.md` card in the middle, surrounded by six labeled modules:

1. 触发与编排
   - 何时激活
   - 任务范围
   - 分支流程
2. 执行生命周期
   - 环境准备
   - 工具 / 脚本
   - 输入收集
   - 输出格式
3. 输出质量
   - 检查清单
   - 验证命令
   - 失败恢复
4. 行为约束
   - 禁止动作
   - 最佳实践
   - 权限边界
5. 领域知识
   - 概念 / 机制
   - 默认值 / 阈值
   - references
6. 人机协同
   - 何时询问用户
   - 何时移交专家
   - 多 agent handoff

Bottom strip: `不是 Prompt，而是 Agent 可执行的上下文包`.

## Visual style

- Use six compact cards around a central document illustration.
- Use concise Chinese labels.
- Avoid long paragraphs.
- Make it look like a review checklist rather than a decorative infographic.

## Data fidelity

The six categories should match the paper's taxonomy: scoping/orchestrating, execution lifecycle, output quality, agent conduct, domain knowledge, user/agent coordination.
