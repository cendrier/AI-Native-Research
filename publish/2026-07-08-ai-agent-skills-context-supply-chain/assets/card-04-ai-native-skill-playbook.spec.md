# Visual Card 04 — AI Native Skill Playbook

## Purpose

Give internal teams a practical operating model for treating agent skills as governed engineering assets.

## Layout

Create a 2x3 grid titled `内部 Skill 治理六件事`:

1. Registry
   - 统一目录
   - owner / version / changelog
   - 适用仓库
2. Review
   - activation criteria
   - allowed tools
   - human confirmation gates
3. Validation
   - scripts 可运行
   - references 有效
   - 输出有检查清单
4. Drift Control
   - 上游更新提醒
   - stale skill count
   - broken reference scan
5. Runtime Observability
   - activation precision
   - manual override rate
   - incident-linked skill
6. Knowledge Architecture
   - 行为契约模板化
   - 项目绑定独立配置
   - 领域知识 source of truth

Bottom line:

> 模型能力 × 可治理上下文 = 可复制的 AI Native Engineering 能力

## Visual style

- Clean internal playbook style.
- Use small icons for registry, review, validation, drift, telemetry, knowledge.
- Avoid abstract AI imagery.
- Make it suitable as a reusable team checklist.

## Data fidelity

This is an interpretation card derived from the article's practical takeaways, not a direct source figure. Do not attach paper citations inside the card.
