# Card 03 — AI Coding 不能只看 Velocity

## Goal

Turn the paper's measurement framing into an operational metric set for AI Native Engineering teams.

## Recommended layout

Use a dashboard-style card with three metric groups:

### 1. Velocity
- AI-assisted PR share
- PR throughput
- cycle time
- review queue time
- deployment frequency

### 2. Complexity / maintainability
- cognitive complexity
- cyclomatic complexity
- change size
- test coverage
- refactoring budget
- ownership risk

### 3. Participation health
- newcomer / new-member inflow
- first PR response time
- first PR merge rate
- 3 / 6 month retention
- starter-task supply
- mentor / reviewer interaction

## Main headline

**AI Native 指标 = 速度 × 可维护性 × 人才入口**

## Visual details

- Make it look like a compact engineering health dashboard.
- Use three equal cards or columns.
- Add a footer line: “不要让 productivity dashboard 遮住 newcomer pipeline”.
- Keep Chinese labels primary; preserve English metric names where they are standard.
