# Card 02 — 复杂度上升 ≠ 新人被挤出

## Goal

Communicate the central result: the complexity mechanism is real, but it does not translate into measurable newcomer crowd-out in the observed established OSS projects.

## Recommended layout

Use a split-card design with a center connector labelled **Decoupling**.

Left side: **复杂度机制存在**
- Python cognitive complexity: **+11%**
- Cross-language cyclomatic complexity: **+3–4%**
- Interpretation: code can become harder to read and reason about

Right side: **新人 pipeline 未显著下降**
- Newcomer inflow: no significant decline
- Onboarding / first PR handling: no systematic worsening
- Retention: unchanged
- Beginner-task supply: no observed decline, but sparse signal

Center connector:
- **Mechanism ≠ outcome**
- Complexity increased, but no detectable crowd-out in this sample

## Main headline

**AI Agent 让代码更复杂，但没有自动挤走新人**

## Visual details

- Use a balanced two-column comparison.
- Emphasize nuance; do not make it look like “AI agents are harmless”.
- Include a small caveat ribbon: “established OSS projects; visible adoption signal”.
- Use normal spacing and avoid too many footnotes.

## Insert location

After the first paragraph of `核心洞察`.
