# Card 02 Spec — Agentic adoption 的三层强度

## Purpose

Summarize the most important metrics from the paper and show why active-user adoption alone is insufficient.

## Style

Follow `docs/AI_NATIVE_RESEARCH_VISUAL_STYLE_V1.md`. Use AI Native Research V1. Clean, analytical, premium research card.

## Canvas

1600 × 900 px, horizontal 16:9.

## Layout

Header zone:
- Series label: AI Native Engineering · Evidence Map
- Main title: Agentic adoption 的三层强度
- Subtitle: 不只看“有多少人用”，还要看“多少工作被系统执行”

Main content zone: three horizontal bands or stacked panels.

Band 1: Adoption breadth / 谁开始使用
- 指标：28-day active users using Codex
- Individual: 0.7%
- Organizational: 17.3%
- OpenAI: 97.9%
- Interpretation: 使用面差异巨大，组织环境决定扩散速度。

Band 2: Usage intensity / 工作量是否迁移
- 指标：Codex share of output tokens across Codex + ChatGPT
- Individual: 16.5%
- Organizational: 63.3%
- OpenAI: 99.8%
- Interpretation: 一旦进入组织流程，agentic interface 会承接主要输出。

Band 3: Workflow depth / 是否开始重构工作方式
- 复杂任务：8h+ human-time tasks from 2.1% → 25.6%
- 并行 agent：OpenAI 28.6% users managed 5+ concurrent agents
- Skills：active users using any skill reached 26.6%; OpenAI 96.2%
- Interpretation: adoption depth = complexity + runtime + concurrency + reuse。

Evidence / metric chips:
- Period: H1 2026
- Source: The Shift to Agentic AI: Evidence from Codex
- Data populations: Individual / Organizational / OpenAI workers

Takeaway zone:
- 衡量 Agentic AI，不要只问“用了没有”，要问“工作是否被委托、并行、复用、交付”。

## Visual notes

Use a clean data-dashboard style. Each band should have a short label, 3 metrics, and a one-line interpretation. Avoid dense charts; this card is a synthesized metric map, not a reproduction of the paper figure.
