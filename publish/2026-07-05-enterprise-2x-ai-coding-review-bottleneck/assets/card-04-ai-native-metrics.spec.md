# Card 04 — AI Native Engineering 指标栈

## Purpose

Translate the paper into a practical measurement system for internal AI coding rollout.

## Suggested layout

Stacked layers from bottom to top:

1. Adoption
   - eligible users
   - first use
   - weekly active days
   - accumulated tool experience
2. Throughput
   - agent-assisted PR
   - merged PR
   - PR cycle time
3. Review capacity
   - review queue time
   - human review ratio
   - automated review coverage
   - substantive comments
   - reviewer load
4. Quality / risk
   - change failure rate
   - revert / rollback
   - incident
   - security finding
   - maintainability
5. Learning system
   - runbook
   - checklist
   - eval
   - reusable skills
   - failure notes

Main headline:

> 从“AI 多写代码”升级到“组织能安全吸收更多变更”。

## Visual style

Use a clean layered architecture diagram. Make it feel like an operating model, not a KPI dashboard.
