# Card 04 — AI Code Trust Metrics

## Purpose

Turn the article into an actionable measurement dashboard for engineering leaders.

## Visual layout

Create a dashboard-style card with four metric groups.

### 1. Adoption / Usage

- AI tool active days
- AI-assisted PR share
- agent-generated change volume

### 2. Provenance / Traceability

- provenance coverage
- model/tool metadata completeness
- incident traceability rate

### 3. Verification / Quality

- independent verification pass rate
- AI review precision / recall
- high-risk path manual review rate
- policy gate failure reasons

### 4. Delivery / Sustainability

- review queue time
- rollback / revert rate
- change failure rate
- maintainability signals
- reviewer load

## Required Chinese title

**不要只看 PR 多了：看 Trust Metrics**

## Required subtitle

AI Native Engineering 的成熟度 = 速度 × 可验证性 × 可追溯性 × 责任清晰度。

## Design notes

- Use four quadrants with compact metric chips.
- Include a small formula or footer: "Speed without traceability = unmanaged risk".
- Keep card readable; no more than 16 metric chips.
