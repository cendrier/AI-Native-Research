# Card 01 — 从写得快到管得住

## Purpose

Opening summary card for the article. Show the shift from AI coding speed to governance gap.

## Visual layout

Use a left-to-right flow with three stages:

1. **Code Generation 加速**
   - 78% developers write code faster
   - 73% perceive higher code quality
   - Multiple AI coding tools in active use

2. **Delivery 没有同步加速**
   - 79% say software delivery has not accelerated at the same pace
   - Bottleneck moves downstream
   - More time spent on review, validation, governance

3. **Trust / Governance 成为新控制面**
   - 85% point to review & validation as biggest constraint
   - Provenance, traceability, accountability become platform capabilities

## Required Chinese title

**从 Code Speed 到 Governance Gap**

## Required subtitle

AI coding 的问题不再是“能不能写”，而是“能不能解释、验证、追溯、负责”。

## Design notes

- 16:9 Feishu-friendly card.
- Light background, clear arrows, three cards or columns.
- Use icons only if subtle: lightning for speed, queue/checklist for delivery, shield/ledger for governance.
- Keep text concise; do not include source URLs on the card.

## Data labels

- 78% 写代码更快
- 73% 认为质量提高
- 79% delivery 未同步加速
- 85% 最大瓶颈：review / validation
